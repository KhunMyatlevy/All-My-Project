Django Social Media Clone

I built a social media clone using Django REST Framework for the backend and JavaScript to consume the API on the frontend. The project includes user, post, and comment models, providing core social media functionality. Authentication is implemented using a token-based system, ensuring secure access. Key features include role-based permissions, allowing only the owner of a post or comment to edit or delete it, and restricting access to posts to logged-in users. This project demonstrates an effective combination of backend API design and frontend integration for dynamic and interactive functionality.

Key Features

  • RESTful API: Built using Django REST Framework to provide scalable and maintainable backend services.
  • Frontend Integration: JavaScript-based frontend that consumes the API for seamless interaction.
  • Authentication: Token-based authentication to ensure secure access for users.
  • Role-Based Permissions: Only post or comment owners can edit or delete their content.
  • Access Control: Posts are accessible only to logged-in users, enhancing privacy and security.

Technologies Used

  • Python: The primary programming language used for developing the backend API with Django REST Framework.
  • Django REST Framework: Facilitates the creation of scalable and maintainable RESTful APIs for the backend.
  • PostgreSQL: A robust relational database used to store and manage user, post, and comment data securely.
  • JavaScript: Utilized for consuming the API and adding interactivity on the client side.
  • HTML: Used to structure the content of the web pages and create a clean and organized layout.
  • CSS: Employed for styling the frontend, ensuring a visually appealing and user-friendly design.
  • Git: Version control system used for tracking changes in the codebase and collaborating effectively.
  • GitHub: The platform used for hosting the project repository, enabling seamless access and version management.
